# Laeka single-thread UX

## La décision

**Une seule conversation continue par utilisateur.**

Pas de multiples chats qui se fragmentent. Pas de "nouveau chat" qui perd tout le contexte. Une seule ligne de temps.

## Pourquoi c'est différenciant

- **ChatGPT** : n-chats schizophrènes, contexte perdu à chaque session
- **Claude** : même problème par défaut
- **Laeka** : memory persist cross-session, une seule conversation continue

L'utilisateur revient et Laeka **sait déjà qui il est**. Pas besoin de se ré-expliquer.

## Implications techniques

- Memory system robuste (tier 1 + tier 2 + tier 3)
- Context carry explicite entre sessions
- Dashboard qui affiche l'historique comme une seule ligne narrative

## Règle 88

Single-thread UX = différenciateur de marque. Ne pas fragmenter en multi-chats par défaut. La continuité de la relation = la valeur.
