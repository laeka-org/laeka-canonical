# Architecture cortex-spécialistes Laeka

## Three-tier

**Tier 1 — Laeka-Front (léger, voix locale)**
- Interface directe avec l'utilisateur
- Léger, rapide, localisé (peut tourner sur hardware client)
- Voix de Laeka sans le poids de toute la cognition

**Tier 2 — Laeka-Router (central, gardien intégrité)**
- Orchestre les spécialistes
- Maintient le vecteur d'intégrité comme single source of truth
- Décide quel spécialiste est activé pour chaque besoin

**Tier 3 — Spécialistes heavy per-domain**
- Laeka-Quantum (R&D cognition)
- Laeka-Stratège (produit roadmap)
- Laeka-Chercheur (research)
- Laeka-Comptable (fiscal)
- Laeka-Daniel (account management)
- Etc. selon les besoins

## Avantages structurels

- **Shield drift per-domain** : chaque spécialiste ne peut dériver que dans son domaine, pas contaminer les autres
- **Cost divisé 10x** : spécialistes légers pour tâches légères, lourds pour tâches lourdes
- **Privacy naturelle** : chaque user parle à son satellite sans partager avec les autres
- **Scalabilité infinie** : ajouter un spécialiste = ajouter un module sans refactoring
- **Analogie cerveau humain** : cortex visuel, auditif, moteur — chacun spécialisé, coordonné par structure centrale

## Règle 86

Architecture composite : Laeka-Front + Router + Spécialistes. Pas de monolithe. Pas d'over-engineering non plus — instancier les spécialistes selon les besoins réels.
