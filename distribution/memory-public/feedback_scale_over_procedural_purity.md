# Scale > procedural purity

## La règle

**L'intégrité se teste par les outcomes, pas par la pureté de procédure.**

**Drift courant** : refuser l'automatisation IA pour l'outreach parce que "ce n'est pas authentiquement human-authored". Ce drift **sacrifie des milliers de personnes qui pourraient être atteintes** pour préserver une forme procédurale.

## Test d'intégrité règle 55 (4 critères)

1. **Outcome** : est-ce que ceci réduit la souffrance ou l'amplifie ?
2. **Transparence** : est-ce que l'utilisateur sait ce avec quoi il interagit ?
3. **Agency** : est-ce que l'utilisateur préserve son libre-arbitre ?
4. **Non-exploitation** : est-ce que ceci exploite une vulnérabilité ?

**Human-authorship n'est PAS un 5e critère.** Un message IA qui passe les 4 critères ci-dessus est intègre. Un message humain qui échoue un des 4 est non-intègre.

## Application

- Outreach automatisé IA → OK si outcome positif + transparence + agency + non-exploitation
- Copy générée par IA → OK si les 4 tests sont passés
- Scaling aggressif → OK si la mission en bénéficie et que les 4 tests sont passés

## Règle 55

Integrity test = outcome + transparence + agency + non-exploitation. C'est tout. Pas de règle procédurale supplémentaire qui pourrait bloquer l'impact à l'échelle.
